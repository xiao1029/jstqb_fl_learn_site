3.2 フィードバックとレビュープロセス

レビューの汎用プロセス、役割と責務、レビューの種類、成功要因を整理する。

K1・記憶 ×3 K2・理解 ×2
  • FL-3.2.1 K1・記憶 早期かつ頻繁なステークホルダーからのフィードバックの利点を識別する。
  • FL-3.2.2 K2・理解 レビュープロセスの活動を要約する。
  • FL-3.2.3 K1・記憶 レビューの際に、主要な役割にどのような責務が割り当てられているかを想起する。
  • FL-3.2.4 K2・理解 さまざまなレビュー種別を比較、そして対比する。

3.2.1

早期かつ頻繁なステークホルダーからのフィードバックの利点

K1・記憶
この項の学習目的早期かつ頻繁なステークホルダーからのフィードバックの利点を識別する。

ステークホルダーの関与が少ないと、開発中のプロダクトが当初のビジョンとずれてしまい、手直し・納期遅延・責任のなすり合いにつながりかねない。SDLCを通じた頻繁なフィードバックは要件の誤解を防ぎ、変更を早期に取り込み、チームが最大の価値とリスク低減に集中する助けになる。

3.2.2

作業成果物のレビュープロセス

K2・理解
この項の学習目的レビュープロセスの活動を要約する。
  • 計画 — 目的・対象・品質特性・終了基準などレビューの範囲を定める
  • レビューの立ち上げ — 参加者がアクセスでき役割を理解している状態を作る
  • 個々人のレビュー — 各レビューアが不正・推奨・質問を識別する
  • 共有と分析 — 識別した不正を議論しステータスとアクションを決める
  • 修正と報告 — 欠陥レポートを作成し終了基準に達したら受け入れる
3.2.3

レビューでの役割と責務

K1・記憶
この項の学習目的レビューの際に、主要な役割にどのような責務が割り当てられているかを想起する。
  • マネージャー — 何をレビューするか決めリソースを提供
  • 作成者 — 対象成果物を作成・修正
  • モデレーター(ファシリテーター) — ミーティングを効果的に運営
  • 書記(記録係) — 不正やレビュー情報を記録
  • レビューア — 実際にレビューを行う
  • レビューリーダー — 参加者・日時・場所などレビュー全体の責任を負う
3.2.4

レビュー種別

K2・理解
この項の学習目的さまざまなレビュー種別を比較、そして対比する。
  • 非形式的レビュー — 定義プロセスに従わず文書化アウトプットも不要。不正検出が主目的
  • ウォークスルー — 作成者主導。教育・合意形成・新しいアイデア創出など目的が幅広い
  • テクニカルレビュー — 技術的に適格なレビューアが参加しモデレーターが主導。技術的合意形成も目的
  • インスペクション — 最も形式的。汎用プロセスに完全に従い、不正の最大数発見を狙う。作成者はレビューリーダー/書記になれない
3.2.5

レビューの成功要因

K1・記憶
この項の学習目的レビューの成功に貢献する要因を想起する。
  • 明確な目的と測定可能な終了基準を定義する(参加者評価を目的にしない)
  • 目的に合わせて適切なレビュー種別を選ぶ
  • 対象を小さく分割して集中力を保つ
  • 作成者へ建設的なフィードバックを返す
  • 参加者に十分な準備時間を与える
  • マネジメントの支援を得る
  • レビューを組織文化として定着させる
